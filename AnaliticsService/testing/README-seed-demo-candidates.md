# Демо-данные для аналитики (10 «кандидатов» / комнат)

## Как заполнено

Скрипт `seed_ten_demo_candidates_api.py` вызывает **HTTP API** сервиса аналитики (`POST /history`, метрики, `POST /interviewer-assessment`) и создаёт **10 комнат** с историей сессии, заметками (имя кандидата в тексте) и оценкой. Данные попадают в **БД аналитики** на сервере.

Прямое подключение к PostgreSQL с произвольной машины может быть запрещено (`pg_hba.conf`) или неверны учётные данные — поэтому основной способ сидирования — **API**.

## Запуск

```bash
cd AnaliticsService
python testing/seed_ten_demo_candidates_api.py
```

Другой хост:

```bash
set API_BASE_URL=http://127.0.0.1:8000
python testing/seed_ten_demo_candidates_api.py
```

## Проверка во фронте

После входа как интервьюер откройте отчёт, подставив UUID, например:

`/interviewer/sessions/56b1cc8a-2b40-5a77-92ae-083ec1c5768d/report`

`GET /ai-summary` при сидировании **не** вызывается (нужен GigaChat); его можно дернуть вручную из Swagger после просмотра отчёта.

## Безопасность

Не коммитьте пароли БД в репозиторий. Если пароль передавался в открытом виде — **смените его** на сервере.
