# K1 Typing Backend (Core Service)

Основной бэкенд-сервис для платформы "NEO Coding Board" (контейнер `java-business-app`).

## Установка и настройка

Все настройки и переменные окружения, необходимые для запуска базы данных и приложения, подготовлены в файле `.env.origin`.

> [!IMPORTANT]
> Файл `.env.origin` является шаблоном. Чтобы всё заработало, необходимо создать на его основе файл `.env` с аналогичными параметрами:
>
> ```bash
> cp .env.origin .env
> ```

При необходимости вы можете изменить параметры в файле `.env` (например, порты или данные для подключения к базе) до старта приложения.

Кроме того, для корректной работы системы авторизации требуются RSA-ключи. В директории `core-service/app/src/main/resources/keys/` находятся файлы-шаблоны: `private_key.pem.origin` и `public_key.pem.origin`.

> [!IMPORTANT]
> Эти файлы необходимо скопировать без расширения `.origin` и заполнить их содержимое реальными RSA-ключами (приватным и публичным соответственно).
>
> Перейдите по указанному пути и скопируйте файлы:
>
> ```bash
> cd app/src/main/resources/keys/
> cp private_key.pem.origin private_key.pem
> cp public_key.pem.origin public_key.pem
> ```
>
> После этого откройте созданные файлы `private_key.pem` и `public_key.pem` и вставьте туда ваши валидные ключи.

## Запуск проекта

1. Убедитесь, что вы находитесь в директории `core-service` в терминале:

   ```bash
   cd core-service
   ```

2. Запустите сборку и старт контейнеров с помощью Docker Compose в фоновом режиме:

   ```bash
   docker-compose up --build -d
   ```

3. Чтобы посмотреть логи и убедиться, что приложение стартовало успешно:

   ```bash
   docker-compose logs -f app
   ```

   *(Для выхода из режима просмотра логов нажмите `Ctrl+C`)*

4. **Остановка проекта:**
   Чтобы остановить работу контейнеров, выполните:

   ```bash
   docker-compose down
   ```

   Если вы хотите остановить контейнеры и удалить все сохраненные данные (volume базы данных), добавьте флаг `-v`:

   ```bash
   docker-compose down -v
   ```

## Доступ к приложению

После успешного старта API приложения будет доступно по адресу:

- **<http://localhost:8080>**

Документация (Swagger UI) доступна по ссылке:

- **<http://localhost:8080/swagger-ui/index.html>**
